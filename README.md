# 📊 Task Tracker - Система управления проектами

Современное веб-приложение для управления проектами и задачами с возможностью экспорта и импорта данных. Построено на Flask и SQLite.

- ✅ Редактирование всех параметров задачи

## 🚀 Быстрый старт

### 1. Установка

#### Клонирование репозитория
```bash
git clone <url-репозитория>
cd task_tracker
```

Или скачайте файлы вручную:

Создайте следующую структуру папок:

```
task_tracker/
├── app.py
├── templates/
│   ├── base.html
│   ├── index.html
│   └── edit_task.html
└── README.md
```

2. Установка зависимостей

```bash
pip install flask
```

Или если есть requirements.txt:

```bash
pip install -r requirements.txt
```

3. Запуск приложения

На macOS / Linux:

```bash
cd task_tracker
python3 app.py
```

На Windows:

```cmd
cd task_tracker
python app.py
```

4. Открытие в браузере

После запуска откройте браузер и перейдите по адресу:

```
http://127.0.0.1:5000
```

📖 Руководство пользователя

Основные операции

Создание проекта

1. Нажмите кнопку "Новый проект"
2. Введите название проекта
3. Выберите менеджера из списка
4. Нажмите "Создать проект"

Создание задачи

1. Нажмите кнопку "Новая задача"
2. Выберите проект
3. Введите описание задачи
4. Назначьте исполнителя (опционально)
5. Выберите приоритет
6. Установите дедлайн (опционально)
7. Нажмите "Создать задачу"

Изменение статуса задачи

Используйте кнопки To Do, In Progress, Done рядом с задачей:

· При переводе в Done автоматически устанавливается дата завершения
· При возврате из Done дата завершения очищается

Редактирование задачи

1. Нажмите кнопку ✏️ (карандаш) рядом с задачей
2. Измените необходимые параметры
3. Нажмите "Сохранить изменения"

Удаление задачи/проекта

· Нажмите кнопку 🗑️ (корзина) для удаления задачи
· Подтвердите действие в диалоговом окне
· Внимание: при удалении проекта удаляются все связанные задачи

Фильтрация задач

Используйте выпадающие списки в панели фильтрации:

· Исполнитель - фильтр по конкретному сотруднику
· Приоритет - фильтр по уровню важности
· Статус - фильтр по текущему состоянию

Нажмите "Сбросить фильтры" для отображения всех задач.

Экспорт данных

Экспорт задач

1. Нажмите кнопку "Экспорт" в верхней панели
2. Выберите "Экспорт задач (CSV)"
3. Файл будет автоматически скачан

Экспорт проектов

1. Нажмите "Экспорт" → "Экспорт проектов (CSV)"
2. Получите файл со статистикой по проектам

Полный экспорт

1. Нажмите "Экспорт" → "Полный экспорт (CSV)"
2. Все данные (пользователи, проекты, задачи) в одном файле
3. Идеально для резервного копирования

Импорт данных

Подготовка CSV файла

1. Скачайте шаблон: "Экспорт" → "Скачать шаблон для импорта"
2. Откройте файл в Excel или Google Sheets
3. Заполните данные по образцу:

```csv
Проект,Описание,Исполнитель,Приоритет,Статус,Дедлайн
Разработка CRM,Создать API,Иванов Иван,High,To Do,2024-04-15
Мобильное приложение,Дизайн экрана,Петров Петр,Medium,In Progress,2024-04-20
```

Обязательные поля: Проект, Описание
Опциональные поля: Исполнитель, Приоритет, Статус, Дедлайн

Допустимые значения:

· Приоритет: Low, Medium, High
· Статус: To Do, In Progress, Done
· Дедлайн: YYYY-MM-DD (например, 2024-12-31)

Загрузка файла

1. Нажмите кнопку "Импорт CSV"
2. Выберите подготовленный файл
3. Нажмите "Загрузить и импортировать"
4. Проверьте сообщения о результате импорта

Важные замечания:

· Проекты создаются автоматически, если не существуют
· Исполнители создаются при необходимости с ролью "Исполнитель"
· При ошибках в строках, корректные строки всё равно импортируются
· Выводится статистика успешно импортированных записей

🗄️ Структура базы данных

База данных создается автоматически при первом запуске в файле tracker.db.

Таблица Users (Пользователи)

Поле Тип Описание
id INTEGER Уникальный идентификатор
name TEXT ФИО пользователя
role TEXT Роль (Менеджер/Исполнитель)

Таблица Projects (Проекты)

Поле Тип Описание
id INTEGER Уникальный идентификатор
title TEXT Название проекта
manager_id INTEGER ID менеджера (внешний ключ)

Таблица Tasks (Задачи)

Поле Тип Описание
id INTEGER Уникальный идентификатор
project_id INTEGER ID проекта (внешний ключ)
description TEXT Описание задачи
performer_id INTEGER ID исполнителя (внешний ключ)
priority TEXT Приоритет (Low/Medium/High)
status TEXT Статус (To Do/In Progress/Done)
created_at DATE Дата создания
deadline DATE Дедлайн
finished_at DATE Дата завершения

❗ Часто задаваемые вопросы (FAQ)

Q: Где хранятся данные?
A: Все данные хранятся локально в файле tracker.db в папке приложения.

Q: Можно ли использовать приложение в команде?
A: Приложение однопользовательское. Для командной работы требуется доработка с использованием серверной СУБД.

Q: Как перенести данные на другой компьютер?
A: Экспортируйте данные в CSV, перенесите файл и импортируйте на новом месте. Или скопируйте файл tracker.db.

Q: Поддерживается ли кириллица в CSV?
A: Да, используется кодировка UTF-8 с BOM для корректной работы с кириллицей в Excel.

Q: Как создать резервную копию?
A: Используйте "Полный экспорт (CSV)" или скопируйте файл tracker.db.

🛠️ Устранение неполадок

Приложение не запускается

```bash
# Проверьте версию Python
python --version  # или python3 --version

# Установите Flask
pip install flask  # или pip3 install flask
```

Порт занят

Если порт 5000 занят, измените в app.py:

```python
app.run(debug=True, host='0.0.0.0', port=5001)  # Измените на другой порт
```

Проблемы с кодировкой в Excel

· Используйте "Сохранить как" в Excel с кодировкой UTF-8
· Открывайте CSV через "Данные" → "Из текста/CSV" в Excel
· Выбирайте разделитель "запятая" и кодировку "UTF-8"

Ошибки при импорте

· Проверьте формат CSV (запятые как разделители)
· Убедитесь, что заголовки точно соответствуют шаблону
· Проверьте даты в формате YYYY-MM-DD
· Убедитесь, что приоритеты указаны корректно (Low/Medium/High)
